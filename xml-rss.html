XML SYNTAX RULES

The syntax rules of XML are very simple and logical. The rules are easy to learn, and easy to use.

Rule 1: The XML Prolog

This is the first line in XML document. It is optional

Example: <?xml version="1.0" encoding="UTF-8"?>

Rule 2: Matching Tags

Every opening tag must have a corresponding closing tag. Tags are case-sensitive.

Example:
<book>
    <title>Sample Book</title>
</book>

Rule 3: Proper Nesting

XML elements must be properly nested within each other, with no overlapping or gaps in the hierarchy.

Example:
<person>
    <name>John Doe</name>
    <address>
        <street>Main St</street>
        <city>Anytown</city>
    </address>
</person>

Rule 4: Attribute Values

Attribute values must be enclosed in single or double quotes.

Example: <student id="123" name="Alice" />

Rule 5: Reserved Characters

Certain characters have special meaning in XML and must be escaped using predefined character entities.

Example: &lt;description&gt;This is an example description. &lt;/description&gt;

Rule 6: Entity References

Use entity references for characters that are not directly supported in the character encoding being used.

Example: <message> Don&apos;t forget to bring your ID card.</message>
